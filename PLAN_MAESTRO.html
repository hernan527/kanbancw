Plan Maestro de Evolución Pro

Documento de arquitectura y roadmap para convertir codigo_kanban en una capa CRM/Kanban avanzada sobre Chatwoot. El análisis refleja el estado real del repositorio: backend compilado disponible, frontend solo como build empaquetado, con puntos de inserción inferidos desde el router y los módulos presentes.

Inserciones Inmediatas

  • Vista expandida de card: el mejor punto es la ruta /conversation/:id, ya existente en el bundle del frontend. La estrategia correcta es extraer esa experiencia a un modal o side-panel reusable invocado desde el board principal en /.
  • Dashboard live: el mejor punto de partida es la ruta /projects-reports, que ya funciona como entrada analítica y reduce costo de adopción. Si se busca separación de dominio, conviene clonarla a una nueva ruta comercial tipo /kanban-reports.
  • Lead scoring con IA: debe estructurarse sobre AICredentials, AIService, los datos de conversación de Chatwoot y la entidad Card. La pieza faltante es el dominio de scoring, no la infraestructura base.
Restricción operativa: el source del frontend no está presente en codigo_frontend; solo existe el build servido por Nginx. El plan asume recuperación del source original o recreación controlada del frontend a partir del contrato backend.

Arquitectura Recomendada para Lead Scoring IA

  • Extender Card con leadScore, leadTemperature, lastScoredAt, scoreBreakdown y aiSummary.
  • Crear LeadScoreConfig por account/funnel con provider, modelo, thresholds, pesos y prompts.
  • Crear LeadScoreRun para auditoría, debugging y explainability.
  • Implementar services/leadScoringService y routes/lead-scoring.
  • Disparadores: alta de conversación, cambio de stage, cambios en custom fields, nuevo mensaje relevante y recálculo manual.
  • Entrega de salida: score 0-100, temperatura, razones, acciones sugeridas y campos a completar.

Fases Recomendadas

  • Fase 1: Vista expandida de card, checklist, custom fields y quick actions.
  • Fase 2: Dashboard live, métricas por funnel y base de stage aging.
  • Fase 3: Lead scoring IA, resumen de conversación y auto-fill.
  • Fase 4: SLA, triggers condicionales y hardening de integraciones.
  • Fase 5: CSV import, timeline completa y optimización analítica.

Dependencias Técnicas

  • Recuperar o recrear el source del frontend para evitar trabajar sobre el bundle compilado.
  • Agregar migraciones Prisma para scoring, notas y eventos temporales.
  • Formalizar contratos de API para board, dashboard y modal CRM.
  • Definir una política de prompts, modelos y costos por funnel para IA.
  • Introducir pruebas de integración sobre Kanban, scoring y automations.
